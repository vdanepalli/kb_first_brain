# Survey123

## Survey123 Web Designer

Two ways to create
- ArcGIS Survey123 **Web Designer**
- **Survey 123 Connect** - XLS Forms (Spreadsheets) - **Case Sensitive**

Permissions: Create, Publish, Survey123 Add on or Field Apps Bundle assigned. 

Survey Header/Title and Survey Name are different. 

Published usinig Web Designer
1. Survey - Survey Name
   1. Form
   2. Feature Layer
   3. Survey_results view
   4. Survey_form view

Published using Survey123 Connect
1. Form
2. Feature Layer
3. Repeat items
4. Result View
5. Web map

Shared Publicly -> anyone can submit; Otherwise -> need to sign into ArcGIS Online Account



Properties: 
1. Question Type
2. Appearance - Geocode, Multiline, Likert, Minimal, Signature ...
3. Option - Default, Required, Read-Only, Calculation, ...

Survey Creation: 
- Plan, Design, Build: Purpose, Questions, Environment, Audience
- Survey Design: Survey Elements, Theme, Appearance, Options. 

Survey Publishing: 
- Preview, Publish, Update

Survey Results:
- View, Analyze, Use 

Owner, Submitter, Viewer

Reports:
- Individual Report
- Summary Report

## Survey123 Connect

1. type
   1. begin group, end group, begin repeat, end repeattext, date, time, integer
   2. select_multiple list_name
   3. select_one list_name 
2. appearance
   1. spinner, compact-2, likert, minimal
3. name, label, list_name, name, label
4. choice_filter (say `itemcuisine = {$cuisine}`)
5. default (say `today()`)   
6. constraint (say `regex(., '\d{3}-\d{3}-\d{4})`)
7. hint
8. bind::type
9. calculation
