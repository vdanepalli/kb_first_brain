## ⚙️ Automation Setup

You can configure a pre-commit hook that automatically updates the UI graph above whenever you commit changes.

**1. Create the hook:**
```bash
touch .git/hooks/pre-commit
```

**2. Paste this script into `.git/hooks/pre-commit`:**
```bash
#!/bin/sh
python3 generate_sitemap.py
git add README.md
```

**3. Make it executable:**
```bash
chmod +x .git/hooks/pre-commit
```

*The sitemap UI will now auto-generate every time you run `git commit`.*

## 📡 Git Cheat Sheet

Quick reference for keeping a clean, linear commit history.

### 1. Attaching a Remote
Bind your local folder to GitHub to sync notes.
```bash
git remote add origin https://github.com/vdanepalli/personal_notes.git
git remote -v
```

### 2. Pushing
Push your local commits to the server.
```bash
git push -u origin main
```

### 3. Amending Commits
Made a typo or forgot a file? Don't clutter your history with a "fix typo" commit. Just bind it to the previous commit.
```bash
git add forgot_this.md
git commit --amend --no-edit
```

### 4. Force Pushing
If you amend a commit *after* you've already pushed it, you must overwrite the remote. Use the safe flag to avoid nuking other data:
```bash
git push --force-with-lease
```

### 5. Pulling Remote Changes
If you drop a note via the GitHub web editor, rebase when pulling to prevent messy merge lines.
```bash
git pull --rebase
```
